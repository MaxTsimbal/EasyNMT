# Task 4C — AI Written Answer Grading

Дата: 2026-07-21

## Мета

Навчити Easy оцінювати зміст і кроки у письмових відповідях 5–11, не
перетворюючи AI на джерело правди для балів, XP або прогресу. Учень має
отримувати часткові бали, конкретну першу помилку, критерії та наступний крок,
а тест повинен завершуватися навіть під час збою зовнішнього провайдера.

## Що реалізовано

### 1. Окремий grading engine

Створено `easynmt_ai/written_grading.py`. Він приймає лише серверні питання
5–11 та формує один обмежений batch-запит для неоднозначних відповідей.
Завдання 12 навмисно виключене: його фото-or-text перевірка належить Task 4D.

### 2. Суворий контракт результату

Модель повинна повернути структурований результат із:

- точним `question_id`;
- балом у межах 2 або 3 доступних балів;
- рівнем упевненості `high`, `medium` або `low`;
- коротким підсумком;
- першою помилкою для неповної відповіді;
- наступним навчальним кроком;
- одним критерієм на кожен доступний бал.

Сума пройдених критеріїв мусить точно дорівнювати запропонованому балу.
Дублікати, пропущені питання, неправильний максимум або суперечливий JSON
відхиляються до запису результату.

### 3. Гібридне оцінювання

Сервер спочатку обчислює детерміновану базову оцінку всіх 12 завдань.
Зовнішній AI отримує лише неоднозначні письмові відповіді 5–11.

- повний серверний бал не витрачає AI-запит;
- `low` confidence повністю ігнорується;
- `medium` confidence може допомогти, але не може забрати вже підтверджений
  сервером частковий бал;
- `high` confidence приймається лише після перевірки ID, рубрики та меж балів;
- завдання 1–4 і 12 залишаються серверними.

### 4. Безпечний життєвий цикл транзакції

Фінальне подання поділено на три фази:

1. коротка транзакція перевіряє власника, token, snapshot і зберігає draft;
2. AI grading виконується після звільнення SQLite writer lock;
3. нова транзакція повторно перевіряє незмінність snapshot та атомарно записує
   attempt, XP і прогрес.

Таким чином повільний мережевий виклик не блокує базу, а підміна тесту між
підготовкою та фіналізацією не може бути збережена.

### 5. Prompt-injection guard

Відповідь учня завжди розглядається як недовірені дані. Спроби наказати
моделі змінити критерії, видати повний бал, розкрити системні інструкції або
ігнорувати rubric не надсилаються grading-моделі. Для такого питання
використовується детермінований fallback із видимою службовою позначкою у
результаті.

### 6. Відмовостійкість

Помилка API, timeout, вимкнений feature flag, неправильна схема або
неповний batch не блокують завершення тесту. Система повертається до
перевіреного локального grader, а серверний цикл XP і прогресу працює далі.

### 7. Навчальний результат

Сторінка результату показує:

- джерело оцінки: сервер, AI-assisted або AI;
- критерій за кожен бал;
- доказ із відповіді учня;
- першу помилку;
- конкретний наступний крок.

Фінальна оцінка, XP і відкриття уроку все одно створюються лише Flask/SQLite.

### 8. Контроль витрат

- не більше одного batch-запиту на фінальну спробу;
- лише неоднозначні питання 5–11;
- повністю підтверджені сервером відповіді пропускають AI;
- окрема модель `OPENAI_GRADING_MODEL`;
- окремий ліміт `OPENAI_WRITTEN_GRADING_MAX_OUTPUT_TOKENS`;
- миттєвий feature flag `OPENAI_WRITTEN_GRADING_ENABLED`.

## Перевірка

Додано регресійні тести для:

- суворої схеми й вибору grading-моделі;
- неправильних максимумів, дублікатів і пропущених ID;
- відбору лише запитань 5–11;
- гарантованого виключення запитання 12;
- узгодження medium/low confidence із серверним балом;
- fallback під час збою провайдера;
- prompt-injection guard;
- пропуску платного виклику для повністю підтверджених відповідей;
- UI-стану перевірки та деталізованого результату;
- відсутності SQLite writer lock під час AI-виклику;
- ідемпотентного повторного submit без другого AI-запиту.

Повна перевірка збірки:

- 180 автоматичних тестів;
- Python `compileall`;
- AST-парсинг 83 Python-файлів;
- JavaScript syntax check для 9 файлів;
- `pip check` без конфліктів залежностей.

## Наступний етап

Task 4D — фінальне завдання 12 з фото або текстом: безпечне завантаження,
перевірка читабельності, аналіз записаних кроків, часткові бали, перша помилка
та можливість повторно завантажити нечітке зображення.
