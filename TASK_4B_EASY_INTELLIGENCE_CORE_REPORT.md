# Task 4B — Easy Intelligence Core

Дата: 2026-07-21

## Мета

Перетворити Easy з одного фіксованого чат-промпта на керований навчальний
інтелект, який розуміє складність запиту, контекст поточного уроку, слабкі
теми учня та бажаний спосіб пояснення. При цьому Flask і SQLite залишаються
єдиним джерелом правди для балів, XP, прогресу, доступу до уроків і результатів
тесту.

## Що реалізовано

### 1. Профілі виконання

Для кожного повідомлення локальний планувальник обирає один із чотирьох
профілів без додаткового платного AI-запиту:

- `fast` — короткі визначення, прості уточнення та режим «Коротко»;
- `balanced` — звичайне навчальне пояснення;
- `deep` — складні задачі, системи, перевірка розв’язання та повторне пояснення;
- `vision` — робота із зображенням.

План також визначає навчальний намір, оцінку складності, потрібну докладність,
рівень reasoning і безпечний ліміт відповіді.

### 2. Розділення моделей за ролями

Додано незалежні змінні середовища:

- `OPENAI_TUTOR_FAST_MODEL`;
- `OPENAI_TUTOR_MODEL`;
- `OPENAI_TUTOR_REASONING_MODEL`;
- `OPENAI_VISION_MODEL`.

За замовчуванням усі вони успадковують `OPENAI_MODEL`, тому оновлення не
змінює витрати без явної конфігурації. Сильнішу модель можна підключити лише
для складних запитів, не оплачуючи її для кожного короткого повідомлення.

### 3. Навчальна пам’ять

Створено таблицю `ai_learner_memory`, яка зберігає лише невеликий набір
педагогічних сигналів для конкретного предмета:

- бажаний стиль: адаптивний, стислий, простий, з підказками або детальний;
- потреба у покроковому поясненні;
- кількість явних сигналів, що попереднє пояснення не допомогло;
- остання тема фокусу.

Пам’ять оновлюється детерміновано з прямих слів учня, наприклад «поясни з
нуля», «коротко», «не кажи відповідь», «я все одно не зрозумів». Для цього не
виконується окремий API-запит.

### 4. Багатший навчальний контекст

Easy тепер отримує серверні дані про:

- цільовий бал;
- діагностичний рівень і результат;
- завершені теми;
- mastery за темами;
- активний навчальний маршрут;
- поточний урок;
- слабкі теми та недавні помилки;
- прогрес, XP і серію навчання.

Контекст обмежений за розміром і не передає секрети або конфігурацію.

### 5. Людяніша відповідь

Оновлений контракт Easy вимагає:

- починати одразу по суті;
- змінювати спосіб пояснення після невдалої спроби;
- показувати, що робимо, навіщо і що отримали;
- у режимі практики вести підказками замість готової відповіді;
- не додавати автоматичні вступи та фінальні фрази.

Додатковий локальний фільтр прибирає типові оболонки на кшталт «Звичайно!» та
«Якщо будуть питання, звертайся», не змінюючи формули й навчальний зміст.
Для streaming-відповіді початок буферизується лише до першого завершеного
речення, щоб шаблонний вступ не встиг з’явитися в інтерфейсі.

### 6. Сумісні reasoning-контролі

Для моделей із підтримкою reasoning провайдер передає обраний рівень
`reasoning.effort` і `text.verbosity`. Для звичайних моделей ці поля не
надсилаються, тому поточна конфігурація `gpt-4o-mini` залишається сумісною.

### 7. Спостережуваність

У метадані повідомлення та відповіді API додано:

- `intelligence_profile`;
- `complexity_score`.

Це дозволяє перевіряти маршрутизацію без запису повного промпта або приватної
історії учня в технічні журнали.

## Межі безпеки

Easy не може самостійно:

- нарахувати XP;
- змінити бал тесту;
- відкрити урок;
- завершити навчальну одиницю;
- змінити ліміти або права доступу.

Ці дії залишаються серверними транзакціями. AI отримує контекст і повертає
пояснення, але не є джерелом правди для прогресу.

## Перевірка

Додано 8 нових тестів для:

- fast/deep/vision маршрутизації;
- збереження педагогічної пам’яті;
- зміни стилю після повторної невдачі;
- безпечного prompt-контракту;
- reasoning-контролів лише для сумісних моделей;
- видалення шаблонних вступів і закінчень.

Повна перевірка збірки:

- 168 автоматичних тестів;
- Python `compileall`;
- перевірка залежностей `pip check`;
- перевірка синтаксису всіх активних Python-модулів.
