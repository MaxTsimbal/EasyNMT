# EasyNMT v1.0 Beta · ручна перевірка

Проходь список на публічному Railway-домені. Для чистого сценарію створи новий
акаунт. Біля помилки записуй URL, дію, час і `X-Request-ID` з DevTools → Network.

## 1. Стан deployment

- Відкрий `/health`: має бути `status: ok`, release `1.0.0-beta.1`.
- Відкрий `/ready`: має бути HTTP 200 і `status: ready`.
- Перевір Railway Variables: OpenAI key, один worker, Volume, secure cookies.

## 2. Новий учень

- Реєстрація email/password без повторного входу.
- Ціль → предмет → час → одноразова діагностика.
- Після онбордингу відкривається кабінет, а не повторне опитування.
- Logout та повторний login зберігають XP і маршрут.
- Окремо перевір Google Login, якщо він налаштований.

## 3. Повний урок

- Відкрий першу тему.
- Під час генерації видно loader, сторінка не виглядає завислою.
- Урок містить мету, теорію, великий приклад, типові помилки, підсумок і три
  самостійні вправи.
- Підказка і розв’язання вправ не відкриті наперед.
- Easy відповідає в контексті уроку й не вигадує прогрес або бали.

## 4. Тест

- 1–4: варіанти відповіді.
- 5–8: короткі письмові відповіді.
- 9–11: повні текстові розв’язання.
- 12: фото, текст або фото + текст.
- Refresh не знищує текстову чернетку.
- Незавершений тест не створює результат.
- Подвійний клік не створює дві спроби.

## 5. Фото №12

- Чітке JPG/PNG/WEBP показує preview і перевіряється.
- Кнопка видалення прибирає preview.
- Нечітке photo-only не перетворюється мовчки на 0/3.
- Текстова альтернатива працює без фото.
- У результаті видно режим і якість розпізнавання, але саме приватне фото не
  зберігається й не відображається.

## 6. Результат і прогрес

- Максимум 24, прохід від 18.
- Кожне завдання має свій бал і пояснення.
- Для 5–11 видно AI grading лише там, де він реально використовувався.
- Після проходження нараховується XP і відкривається наступна тема.
- Повторне відкриття результату не додає XP.
- Невдала спроба дозволяє пройти тест повторно.

## 7. Стійкість

- Відкрий той самий тест у двох вкладках і заверши одну.
- Онови результат, повернися в кабінет, вийди й увійди.
- Перевір телефон: меню, input Easy, quiz navigation, photo chooser, sticky
  submit і результат не перекривають контент.
- У DevTools увімкни Slow 3G: loader і повідомлення залишаються правдивими.

## 8. Backup після acceptance test

У Railway Shell або локально з тим самим Volume:

```text
python -m flask --app app beta check
python -m flask --app app beta backup --reason after-acceptance-test
```

Збірка готова для перших тестових учнів лише коли `/ready` має 200 і весь
основний сценарій проходиться без блокуючої помилки.
