# EasyNMT v1.0 Beta

Це останнє фундаментальне оновлення перед `EasyNMT v1.0 Beta`.

## Головна зміна

Easy більше не розвивається як окремий чат. Він став модулем **AI Викладач** усередині навчальної платформи. EasyNMT передає йому предмет, ціль, прогрес, XP, серію, поточний урок і слабкі теми.

## Що готово

### AI Викладач
- фінальний мобільний і desktop-інтерфейс;
- справжній SSE transport;
- нативні deltas Responses API, коли встановлено ключ;
- локальна та серверна історія;
- синхронізація назв, закріплення, видалення й оцінок відповідей;
- три навчальні режими: пояснення, коротко, практика;
- зупинка генерації, повтор відповіді, копіювання, Markdown і формули;
- завантаження до трьох фото розв’язання;
- безпечний демо-режим без API-ключа.

### OpenAI foundation
Новий пакет `easynmt_ai/` із чітким поділом відповідальності:

```text
easynmt_ai/
├── attachments.py   # перевірка та збереження фото
├── orchestrator.py  # єдина логіка запиту й збереження
├── repository.py    # серверна пам’ять SQLite
├── schemas.py       # типи даних AI-шару
├── service.py       # єдине місце виклику OpenAI SDK
└── prompts/
    └── builder.py   # системний голос і навчальний контекст
```

Жоден інший модуль сайту не повинен напряму викликати OpenAI SDK.

### Серверна пам’ять
Додані таблиці:
- `ai_conversations`;
- `ai_messages`;
- `ai_attachments`;
- `ai_message_feedback`.

Нові повідомлення синхронізуються між браузером і базою. Для наступного запиту бекенд бере історію поточної розмови із серверної бази, а не довіряє лише даним браузера. Це фундамент майбутньої персональної пам’яті, але не «безмежна пам’ять» моделі.

### API
- `GET /api/ai/status`
- `POST /api/ai/attachments`
- `GET /api/ai/conversations`
- `PATCH /api/ai/conversations/<id>`
- `DELETE /api/ai/conversations/<id>`
- `POST /api/ai/messages/<id>/feedback`
- `POST /api/tutor-chat`
- `POST /api/tutor-chat/stream`

## Безпека ключа

`OPENAI_API_KEY` зберігається лише у `.env` локально або в Railway Variables. Файл `.env` не входить у реліз і не повинен потрапляти в GitHub.
