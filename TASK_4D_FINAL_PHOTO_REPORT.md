# Task 4D — Final Photo Solution

Дата: 2026-07-21

## Мета

Зробити лише фінальне завдання №12 мультимодальним: учень може завантажити
фото розв’язання, написати його текстом або використати обидва способи. Фото
має оцінюватися за тими самими трьома балами, не створювати прихованих нулів
під час збою vision-моделі та не передавати моделі контроль над XP чи
прогресом.

## Що реалізовано

### 1. Фото тільки в запитанні №12

Питання 1–11 не мають файлових полів. №12 стало повним фінальним завданням,
де три бали відповідають методу, ключовим крокам і правильній фінальній
відповіді. Текст залишається повноцінною альтернативою фото.

Схему нових тестів піднято до `quiz.v1.5-photo-final`, але стабільний quiz ID
збережено, тому існуючий запис оновлюється на місці без видалення бази.

### 2. Безпечне тимчасове завантаження

Підтримуються PNG, JPG, JPEG і WEBP до 6 МБ. Перед AI-перевіркою сервер:

- перевіряє розширення, MIME, фактичну читабельність і розміри;
- застосовує EXIF-орієнтацію;
- зменшує найбільшу сторону до 2400 px;
- перетворює зображення на JPEG і цим прибирає EXIF та інші метадані;
- використовує тимчасову системну директорію, а не Railway Volume;
- видаляє файл у `finally` одразу після перевірки.

У базі не зберігаються байти фото, локальний шлях або початкове ім’я. До
review потрапляють лише безпечний опис способу подання та обмежена
транскрипція видимих кроків.

### 3. Окремий Easy Vision grader

Створено `easynmt_ai/final_solution_grading.py` зі строгим structured-output
контрактом:

- рівно одне `question_id` і лише номер 12;
- максимум 3 бали;
- три критерії, кількість виконаних критеріїв дорівнює балу;
- `high`, `medium` або `low` confidence;
- `clear`, `partly_readable`, `unreadable` або `not_provided` image quality;
- режим `text`, `photo` або `photo_and_text`;
- коротка транскрипція, підсумок, перша помилка й наступний крок.

Фото й текст учня вважаються недовіреними даними. Інструкції всередині
зображення або відповіді не можуть змінити rubric чи вимагати повний бал.

### 4. Відмовостійке узгодження оцінки

- `high` confidence приймається після суворої серверної валідації;
- `medium` confidence не може забрати вже підтверджений сервером текстовий бал;
- `low` confidence для фото + текст переходить на серверну оцінку тексту;
- нечітке фото без тексту повертає учня до №12;
- недоступна vision-модель без текстового fallback не перетворюється на 0/3;
- текстове №12 працює навіть із вимкненим Easy Vision.

Фінальне збереження спроби, XP і відкриття наступного уроку залишається
атомарною операцією Flask/SQLite після повторної перевірки quiz snapshot.

### 5. Новий інтерфейс

У №12 додано:

- окрему кнопку вибору фото;
- локальний preview без завантаження до завершення тесту;
- перевірку типу й розміру у браузері;
- кнопку видалення фото;
- видимий вибір `фото або текст`;
- правильний лічильник завершення, де фото без тексту зараховує №12 як
  заповнене;
- пояснення, що текстова чернетка зберігається, а фото надсилається лише під
  час фінального submit.

Сторінка результату показує `Easy Vision`, режим `Фото`, `Фото + текст` або
`Текст`, а також якість прочитання без відображення самого приватного фото.

## Контроль витрат

- максимум один окремий vision-запит на одну завершену спробу;
- виклик відбувається лише для №12;
- окрема модель і output-token limit;
- feature flag дозволяє миттєво вимкнути vision grading;
- текстовий fallback не потребує фотоаналізу.

## Перевірка

Додано регресійні тести для:

- обмеження photo grader лише номером 12;
- одного attachment і суворої structured-output схеми;
- суперечливого `submission_mode` та image metadata;
- фото-only оцінки й часткових балів;
- недоступної vision-моделі без прихованого нуля;
- нечіткого фото;
- low-confidence фото + текст fallback;
- re-encode, downscale, очищення EXIF і видалення файлу;
- відхилення не-зображень;
- єдиного photo input у UI;
- vision і photo metadata на сторінці результату.

Повна перевірка збірки:

- 191 автоматичний тест;
- 31 subtest;
- Python `compileall`;
- JavaScript syntax check;
- `pip check` без конфліктів.

## Результат етапу

Task 4D закриває п’ятий і фінальний крок поточного навчального циклу:
результати й прогрес, повноцінні уроки, intelligence core, AI grading 5–11 та
мультимодальне фінальне завдання №12. Наступний великий етап уже має бути не
ще одним локальним модулем, а наскрізним beta-аудитом реального навчання від
реєстрації до завершення теми.
