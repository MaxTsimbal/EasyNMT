# Task 3D.3 — Quiz 409 Stability Recovery Report

## Знайдена причина

Сховище `curriculum_quizzes` вважало шаблон тесту незмінним усередині однієї версії схеми. Якщо урок був повторно згенерований або код тесту змінив його вміст без нового рядка бази, `save_quiz()` кидав `CurriculumQuizPersistenceError`. Маршрут перетворював цю помилку на загальний HTTP 409, тому користувач бачив помилку ще до відкриття тесту.

## Реалізоване виправлення

1. Шаблон тесту тепер оновлюється в тому самому рядку бази після повної перевірки.
2. Первинний `quiz_id` зберігається, тому старі foreign keys сесій і спроб не ламаються.
3. Активні сесії залишаються незмінними, бо мають власний захищений snapshot.
4. Непошкоджена незавершена сесія повторно використовується при refresh.
5. Пошкоджена незавершена сесія видаляється і замінюється новою.
6. Осиротіла позначка `submitted_at` відновлюється, якщо фактичної спроби ще немає.
7. Помилки сховища тепер логуються з маршрутом, типом і деталями та не маскуються під звичайний conflict.
