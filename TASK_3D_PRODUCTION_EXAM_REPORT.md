# Task 3D Production Exam Engine

## Що було не так

Попередня перевірка була технічно надійною, але багато питань просили пояснювати поняття або правило. Для учня це виглядало повторювано, не нагадувало живу екзаменаційну практику, а питання №12 залежало від попереднього завдання.

## Що реалізовано

### 1. Новий англійський exam bank

Додано `easynmt_core/quizzes/english_exam_bank.py` з окремими наборами для всіх 12 англійських тем:

- шість граматичних тем;
- word formation;
- collocations і phrasal verbs;
- gist/detail reading;
- cohesion/inference reading;
- integrated Use of English;
- NMT strategy.

Завдання будуються з перевірених речень, текстів, діалогів і ситуацій. Варіант обирається детерміновано, тому сервер завжди знає ключ і може повторити ту саму спробу.

### 2. Нова структура 12 питань

- **1–4**: contextual multiple choice, 1 бал.
- **5–8**: точна практична відповідь, 2 бали.
- **9–12**: три незалежні компоненти, 3 бали.

У граматичних темах серед практичних вправ є:

- перетворення в заперечення;
- утворення загального питання;
- складання речення з перемішаних слів;
- виправлення помилки;
- переклад;
- мінідіалоги;
- комбіновані завдання з трьох частин.

### 3. Варіанти без конфлікту з чернетками

Номер варіанта залежить від користувача, теми та кількості вже завершених спроб. Через це:

- оновлення сторінки не підміняє питання під уже введеною чернеткою;
- паралельні вкладки бачать той самий набір завдань;
- після відправлення наступна спроба отримує інший набір.

### 4. Оцінювання

Додано режими:

- `exact` для трансформацій, порядку слів, перекладу та коротких речень;
- `rubric` для трьох незалежних частин по одному балу.

Регістр і кінцева пунктуація не впливають на точну відповідь. Майже правильна структура може отримати частковий бал. Ключ і rubric залишаються лише на сервері.

### 5. Easy v3.1

Easy отримує:

- тип навички;
- видиму інструкцію;
- текст завдання;
- reading passage;
- формат відповіді.

Easy не отримує:

- correct answer;
- accepted answers;
- scoring parts;
- grading mode;
- explanation key.

Окремі підказки додані для заперечення, питання, порядку слів, виправлення, перекладу, читання та трьохчастинних вправ.

### 6. UI і результат

Сторінка тесту отримала:

- прогрес заповнення 0/12;
- навігацію по номерах;
- позначення навички;
- окремий блок тексту для reading;
- компактні картки;
- зрозумілий формат 3×1 бал;
- мобільну адаптацію.

Сторінка результату формує до трьох пріоритетів повторення з конкретною дією, наприклад «зроби 5 перетворень у заперечення».

## Сумісність

- Старі snapshots залишаються читабельними завдяки default-полям моделей.
- Ідентичність quiz збережена, а schema піднята до `quiz.v1.4-production-exam`.
- Для математики, української та історії збережено generic builder; питання №12 зроблено незалежним і зрозумілим.
- База даних не потребує нової міграції.

## Перевірка збірки

Task 3D накладено на повну актуальну основу Mentory разом із попередніми Task 3C шарами. Перевірено:

- `141` автоматизований тест пройдено;
- `31` subtest пройдено;
- усі Python-модулі компілюються;
- Jinja-шаблони синтаксично валідні;
- активні JavaScript-файли проходять `node --check`;
- усі 12 англійських тем створюють 12 питань із розподілом `4×1 + 4×2 + 4×3 = 24` бали;
- одна й та сама незавершена спроба відновлює той самий варіант, а наступна завершена спроба отримує інший.
