# Easy Chat v2.0

Статус: повна перед-AI основа інтерфейсу чату.

## Головне

Easy Chat v2.0 більше не є однією сторінкою з полем вводу. Це окремий AI Workspace, підготовлений до наступного етапу: інтеграції OpenAI Responses API, серверної пам’яті та аналізу фото.

## Інтерфейс

- повністю новий повноекранний дизайн у стилі EasyNMT;
- окрема історія розмов із пошуком;
- створення, перемикання, перейменування, закріплення, видалення та експорт чатів;
- компактний режим бокової панелі;
- окремі desktop і mobile layouts;
- новий стартовий екран, повідомлення, AI-card та composer;
- адаптація до мобільної клавіатури через Visual Viewport API;
- safe-area підтримка для мобільних пристроїв;
- анімації з підтримкою prefers-reduced-motion.

## Поведінка

- жодного перезавантаження сторінки під час розмови;
- POST streaming через `/api/tutor-chat/stream` у форматі Server-Sent Events;
- локальний стан «Easy думає» замість глобального loading screen;
- плавна потокова поява тексту;
- зупинка генерації;
- повторна генерація останньої відповіді;
- копіювання повідомлень і code blocks;
- оцінка відповіді кнопками «корисно / потрібно краще»;
- автоскрол та кнопка переходу до нового повідомлення;
- slash-команди `/explain`, `/example`, `/test`, `/mistake`;
- три режими відповіді: пояснення, коротко, практика.

## Форматування

- заголовки;
- абзаци;
- списки;
- цитати;
- таблиці;
- inline code та code blocks;
- безпечні посилання;
- LaTeX через MathJax.

## Збереження

До серверної пам’яті історія зберігається в localStorage окремо для кожного користувача браузера. Схема даних уже розділяє розмови, повідомлення, контекст уроку, оцінки та налаштування.

## Основні файли

- `static/css/easy_chat_v2.css`
- `static/js/easy_chat_v2/storage.js`
- `static/js/easy_chat_v2/markdown.js`
- `static/js/easy_chat_v2/app.js`
- `app.py`
- `ai_service.py`

## Наступний етап

1. підключити справжній OpenAI Responses API streaming;
2. замінити локальну історію серверною пам’яттю;
3. додати завантаження та аналіз фото;
4. підключити контекст уроків, прогресу й помилок;
5. додати перевірку письмових відповідей.
