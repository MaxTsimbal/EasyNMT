# Task 3C.2 Student Clarity Final — звіт

## Мета

Прибрати ситуації, коли учень бачить технічну назву навички замість реального завдання або мусить писати відповідь у стилі AI, щоб отримати бали.

## Реалізація

### Зрозуміла структура питання

Кожне нове питання містить окремі серверні поля:

- `instruction` — конкретна дія учня;
- `task` — речення, задача, текст або ситуація;
- `answer_format` — довжина та форма очікуваної відповіді.

Старе поле `prompt` збережене для сумісності. Старі снапшоти без нових полів продовжують відкриватися.

### Конкретні завдання для всієї платформи

Додано review-bank із 225 завданнями для 75 опублікованих тем:

- математика: 39 тем;
- українська мова: 12 тем;
- історія України: 12 тем;
- англійська мова: 12 тем.

Для кожної теми є три різні завдання з кінцевою відповіддю, коротким міркуванням і способом перевірки. Банк використовується лише тоді, коли приклад уроку є абстрактною назвою методу, а не реальною вправою.

### Людське оцінювання

- питання 5–8 прямо повідомляють, скільки частин треба написати;
- питання 9–11 відділяють бали за результат від бала за пояснення;
- питання 12 просить один конкретний спосіб перевірки;
- правильний результат не обнуляється через відсутність довгого еталонного міркування;
- відповідь своїми словами може отримати повні або часткові бали.

### Contextual Easy

У тесті Easy отримує:

- номер активного питання;
- `instruction`;
- видиме `task`;
- `answer_format`;
- безпечні загальні правила уроку.

Він не отримує правильну відповідь, accepted answers, grading keywords або explanation. Для прохання «поясни простіше» prompt вимагає окремо пояснити, що робити, на що дивитися і як оформити відповідь.

### Інтерфейс

- технічні написи замінено на учнівські;
- блоки питання мають чітку візуальну ієрархію;
- сторінка результату повторює структуру завдання;
- збережено Contextual Easy v3.0.1: плавне друкування, автопрокрутка, Markdown, stop-action і локальний loader замість глобального.

## Перевірка

- Python compileall: успішно;
- unittest: 138 тестів успішно;
- JavaScript syntax check: успішно для contextual Easy, page transitions і markdown renderer;
- task-bank: точно 75 тем і 3 унікальні завдання на кожну;
- public quiz payload: містить зрозумілі поля, але не містить ключа оцінювання;
- legacy quiz snapshot roundtrip: успішно;
- секрети, бази, `.venv`, `.git`, cache-файли не входять до release-архіву.

## Межі пакета

Цей пакет не додає photo grading і не змінює модель curriculum. Він завершує student-facing основу текстових production-тестів та контекстної допомоги Easy перед етапом 4.
